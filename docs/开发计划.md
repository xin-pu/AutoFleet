## 1. 目标与范围

### 1.1 首版（M1～M6）必须交付

- [ ] 单线 SQLite 数据库与领域模型（对齐 Demo 实体）
- [ ] 机台 / 标签 / 规则 CRUD
- [ ] 标签类打分因子 + 完整调度链路（**验收：与 Demo 六场景一致**）
- [ ] 调度 HTTP API（含 `disposition`）+ 幂等 + 日志
- [ ] WPF：基础配置、规则配置、监控看板
- [ ] 机台状态 API 推送 + WPF 人工覆盖
- [ ] Fail 区机台与报废路由规则

### 1.2 首版明确不做

- 动态良率 snapshot / 参数打分（M7/M8）
- 多线中央库
- 脚本化规则引擎

## 3. M3 验收标准（与 Demo 对齐）

引擎迁移时增加 `disposition` 入参与规则触发；六场景结果不变：

| 场景 | disposition | 下一工序 | 预期机台 |
|------|-------------|----------|----------|
| 首次进A K0000 | first_test | A | M-A-02 |
| A通过去B A0001 | pass_to_b | B | M-B-01 |
| A失败复测 | fail_retest | A | M-A-01 |
| A失败换机 | fail_retest | A | M-A-02 |
| A失败去Fail区 | fail_scrap | FAIL | M-FAIL-01 |
| A通过去B K0000 | pass_to_b | B | M-B-02 |
